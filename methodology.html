How it works

Methodology

Walkshed polygons on this site represent the area reachable on foot within a set time limit from a transit station, calculated using real pedestrian network data rather than a simple radius. This page explains how those polygons are built, what data sources are used, and where the current limitations lie.

What is a walkshed?

A walkshed (sometimes called a pedestrian shed or walkable catchment area) is the set of all points reachable from a given location by walking within a defined time or distance threshold. Unlike a simple circular buffer, a walkshed follows the actual street and path network — so barriers like highways, rivers, and dead-end streets are correctly reflected in the shape.

On this site, walksheds are computed per station and represent the area a pedestrian could realistically reach in order to access transit — not the area someone might travel to after alighting.

Key concept

A 10-minute walkshed is not a circle. In a dense grid city it may cover 10+ blocks in every direction; near a highway interchange it may cover just a few streets.

Data sources

Pedestrian network data is sourced from OpenStreetMap (OSM), which provides routable footway, path, and street geometries. Station locations are derived from each agency's official GTFS feed, cross-referenced against OSM node positions where available.

Note

OSM coverage and accuracy varies by city. Urban cores tend to be well-mapped; suburban areas may have incomplete footway data, which can cause walksheds to appear smaller than they really are.

How polygons are computed

Walksheds are precomputed offline using a routing library applied to the OSM pedestrian network. For each station, an isochrone is generated representing all points reachable within the time threshold. The resulting polygon is then simplified and stored as a static GeoJSON file, which is loaded directly in the browser with no server-side processing at runtime.

This static approach keeps the site fast and free to host, at the cost of requiring a manual rebuild whenever underlying data changes.

Current limitation

Walksheds are not updated automatically. They reflect OSM data as of the date noted on each agency page. Significant changes to the street network or station locations may not yet be reflected.

What's not included

Current walksheds do not account for elevation change, pedestrian signal timing, crosswalk availability, or accessibility constraints. They assume a flat, unobstructed walk at a constant speed. Future versions may incorporate slope data and more nuanced pedestrian routing.

Reproducing this work

All source code and GeoJSON data are available on GitHub under an open licence. If you spot an error or want to contribute data for a new agency, contributions are welcome via the project repository.
